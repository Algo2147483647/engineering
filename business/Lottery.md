# Lottery

抽奖是一种用“概率奖励”驱动用户行为的游戏化增长工具：利用不确定性激发用户参与意愿，并将参与过程包装成一个兼具期待感、反馈感与奖励感的互动系统。用户对“可能中奖”的心理预期，会显著放大其参与动力。即使奖品本身价值不高，只要用户在感知上相信存在中奖机会，就能有效提升点击、访问、任务完成率及转化行为。

核心设计问题是：**中奖概率、奖品价值、中奖反馈和成本控制之间如何平衡。**

## 抽奖资格获取

抽奖资格是连接业务目标和用户行为的关键, 通过前置行为获取资格，通过游戏反馈提升动力，通过奖池分层引导用户完成更高价值的业务目标。这样，抽奖才能从单纯的奖励工具，升级为可持续驱动增长的互动机制。 与前置增长玩法的协同: 

- 任务 + 抽奖：用随机奖励降低行动门槛. 最基础、也最常见的组合方式。平台不直接让用户抽奖，而是先让用户完成指定任务，再奖励一次或多次抽奖机会。让用户在玩的过程中自然完成业务指标。用随机奖励降低用户行动门槛，用游戏反馈提升参与动力，用奖品激励引导用户完成业务目标。
- 进度型游戏 + 抽奖, 用累计行为制造目标感. 进度型抽奖强调的不是单次随机，而是让用户看到自己正在接近某个奖励目标。核心是“接近完成”的心理驱动。用户一旦看到自己已经完成了 70%、80%，更容易继续投入。
- 收集 / 积分 + 抽奖：用稀缺道具拉长活动周期. 常见于大型营销活动，例如集五福、集卡片、集勋章、集碎片兑换抽奖机会。但它的复杂度也更高，需要设计卡片概率、稀缺度、交换机制、防作弊规则和活动节奏。
- 裂变 + 任务 + 抽奖：用好友关系放大增长. 社交型抽奖把用户的个人参与行为，扩展为邀请、分享、助力、组队等社交行为。用户为了获得更多中奖机会，把活动传播给更多人。
- 会员 + 抽奖：用身份权益提升用户价值感. 权益型抽奖把用户身份与奖池质量绑定，常用于会员体系、等级体系、积分体系。这类机制的核心是提升身份价值感。身份越高，抽奖权益越好。

**抽奖频次限制**. 抽奖频次限制了用户可以以什么节奏参与抽奖。通过频次限制控制成本，避免奖励被过快消耗.

#### 中奖概率设计

每个奖品的中奖概率是多少?

中奖概率是固定的，还是会根据库存、时间、用户等级动态变化

是否需要保底机制？例如抽满 N 次必中。



放回与无放回抽奖主要对于奖池已经预设好不再增加或调整. 对于可动态调整的奖池, 问题更加复杂. 比如一等价库存固定, 但是保持抽奖过程中保持同一概率, 低价值和空奖的概率和占比按照业务逻辑实时计算.



**放回与无放回抽奖**, 在于每次抽取后，样本空间是否被重置。例如：在固定时间开奖，并从所有报名用户中随机抽出 (N) 名中奖者。这就是典型的**无放回抽奖**。因为中奖名额是有限的，抽出一个中奖者后，剩余中奖名额就会减少，且同一个用户不会被重复抽中。


|          | 放回抽奖                                                     | 无放回抽奖                                                   |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 定义     | 每次抽取后都将奖品放回，下一次抽奖的样本空间不变。           | 每次抽取后奖品不再放回，奖池会随着抽取结果发生变化。         |
| 概率特征 | 每次面对的都是同一个概率实验，各次抽取之间相互独立。每次中奖概率相对固定。 | 各次抽取之间不独立，后续概率受此前结果影响。中奖概率会随着奖池变化而动态变化。 |
| 数学分布 | 二项分布 $P(X=x)=\binom{n}{x}p^x(1-p)^{n-x}$                 | 超几何分布 $P(X=x)=\frac{\binom{K}{x}\binom{N-K}{n-x}}{\binom{N}{n}}$ |
| 适用场景 | 适合独立、大规模、标准化激励场景。低成本、高频次、可规模化发放的奖品，如积分、优惠券、虚拟权益等。更适合“稳定激励”和“大规模参与”。 | 适合有限奖池、强成本控制、稀缺感运营场景。稀缺奖品、高价值奖品、库存有限奖品，如实物大奖、限定权益、限量资格等。更适合“成本锁定”和“稀缺运营”。 |
| 用户感知 | 用户每次参与时面对的中奖概率相对一致，理解成本低。           | 用户可能感知到奖品被抽走后，中奖机会下降。                   |
| 优势     | 激励稳定，体验简单，适合高频参与，不需要用户理解奖池变化。   | 奖品总量和活动成本绝对可控，能够强化稀缺感和紧迫感。成本绝对可控，因为奖品数量固定，不会超发。 |
| 劣势     | 成本在期望上可控，但不是绝对可控。短期内可能出现实际发奖成本超过预算的情况。如果奖品价值较高或中奖概率设计不当，可能导致短期发奖成本过高。 | 可能出现早期用户中奖率较高、后期用户几乎没有中奖机会的问题。需要设计分时段释放奖品、分层奖池、库存保护、保底奖励等机制。 |
| 案例     | 每天签到获得一次抽奖机会, 每次签到都对应一次稳定的激励反馈。 | 限量大奖、限量实物奖品、限量名额抽奖等。                     |




#### 开奖时机

“固定时间开奖”和“即时抽奖”本身并不构成数学意义上的概率差异，而是开奖时机与业务反馈机制的差异。它们会影响概率表达、成本控制、用户心理和运营节奏。

固定时间 / 即时抽奖是“开奖时机”的设计；放回 / 无放回是“概率结构”的设计。

尤其是高价值奖品，如果用户点击按钮后立刻知道结果，容易产生“系统是否真的随机”的疑虑；而固定时间统一开奖，更容易被包装成正式、公平、可审计的活动机制。

|          | 固定时间开奖                                                 | 即时抽奖                                                     |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 定义     | 先形成参与集合，再统一抽取。即先聚集用户，再集中分配有限奖励。 | 每次参与后立刻判定结果。                                     |
| 用户体验 | 用户需要等待开奖，期待感更强。                               | 立刻反馈，刺激感强. 可强化用户行为。                         |
| 抽样方法 | 多为无放回抽样                                               | 可放回，也可无放回                                           |
| 业务价值 | 适合蓄水和造势。用户参与后，在开奖前会持续保持期待感。通常会带来“所有人同池竞争”的公平感：大家先报名，再统一抽取，规则感更强。更适合低频、高价值、强期待的场景。 | 适合即时激励、行为闭环和转化提升。用户完成行为后立刻获得反馈，更容易形成即时激励。 |
| 劣势     | 反馈较慢，转化闭环较弱，不适合单纯追求高频行为反馈的场景。   | 因为反馈速度快，用户和黑产都可以快速试错，依赖实时风控。     |
| 典型场景 | 报名抽奖、预约抽签、高价值大奖                               | 签到抽奖、任务抽奖、转盘、开宝箱                             |
